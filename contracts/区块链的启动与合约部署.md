# 区块链的启动与合约部署

## 环境要求

- Node.js v18+（推荐 v18.19.1，README 指定版本）
- npm v8+
- 操作系统：Windows / macOS / Linux

---

## 首次使用：安装依赖（只需做一次）

```cmd
cd "D:\BaiduNetdiskDownload\06-LauncPad-farm+IDO\C2E-LaunchPad（后台golang版本）\c2n-contracts"
npm install
```

同时将环境变量文件复制一份：

```cmd
copy .env.example .env
```

---

## 每次启动流程

### 第一步：启动本地区块链节点

打开一个 CMD 窗口，执行以下命令：

```cmd
cd "D:\BaiduNetdiskDownload\06-LauncPad-farm+IDO\C2E-LaunchPad（后台golang版本）\c2n-contracts"
npx hardhat node
```

启动成功后会输出本地节点地址和 20 个测试账户：

### 第二步：部署所有合约

```cmd
cd "D:\BaiduNetdiskDownload\06-LauncPad-farm+IDO\C2E-LaunchPad（后台golang版本）\c2n-contracts"
npx hardhat run --network local scripts/deployment/deploy_all.js
```

## 部署的合约说明

| 合约 | 说明 |
|------|------|
| C2N Token | 平台核心代币（ERC20） |
| Airdrop | 空投合约，向用户发放 C2N |
| FarmingC2N | 流动性挖矿合约，质押 LP 赚取 C2N |
| Admin | 管理员权限控制合约 |
| SalesFactory | IDO 销售工厂，创建每次 IDO 子合约 |
| AllocationStaking | 分配质押合约，质押 C2N 获得 IDO 参与资格 |
| MOCK-TOKEN | 模拟项目代币（IDO 中被出售的新币） |
| C2NSale | 具体的 IDO 销售合约 |

合约地址会自动保存到 `deployments/contract-addresses.json`。

---

## 注意事项

- 每次重启 `npx hardhat node` 后，链上数据会**完全清空**，需要重新执行第二步部署合约。
- 两个 CMD 窗口必须同时开着：一个跑节点，一个执行部署。
- 部署脚本路径：`scripts/deployment/deploy_all.js`
