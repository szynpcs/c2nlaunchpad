# 后端启动指南

## 环境要求

- Docker Desktop（已安装并运行）
- 本地以太坊节点（Hardhat 或 Anvil，地址 `http://127.0.0.1:8545`）

---

## 方式一：Docker Compose 启动（推荐）

自动启动 Go 后端 + MySQL 数据库，无需手动配置数据库。

```bash
cd c2e-be-golang/deployment
docker-compose up -d
```

启动后包含两个容器：

| 容器名 | 说明 | 端口 |
|--------|------|------|
| `portal-api` | Go 后端服务 | 8080 |
| `brewery-mysql` | MySQL 5.7 数据库 | 3300→3306 |

> **注意：** MySQL 主机端口为 3300（非默认 3306），原因是 Windows Hyper-V 将 3307-4386 范围预留，导致 Docker 无法绑定 3307。
> 如需修改映射端口，修改 `deployment/docker-compose.yml` 中的 `"3300:3306"`。

---

## 方式二：本地直接运行

**前提条件：**
- Go 1.23+
- MySQL 运行在 `localhost:3306`，数据库名 `brewery`，root 密码 `123456`

```bash
cd c2e-be-golang
go run main.go
```

---

## 完整 IDO 流程（三个服务全部启动）

```bash
# 1. 启动本地以太坊节点
cd c2n-contracts
npx hardhat node

# 2. 部署合约（新开终端）
cd c2n-contracts
make ido

# 3. 启动后端（新开终端）
cd c2e-be-golang/deployment
docker-compose up -d

# 4. 启动前端（新开终端）
cd c2n-fe
yarn dev
```

---

## 验证启动成功

```bash
# 查看容器状态
docker ps --filter "name=portal-api" --filter "name=brewery-mysql"

# 查看后端日志
docker logs portal-api

# 调用健康检查接口
curl http://localhost:8080/boba/hello
# 期望返回：Hello, World!
```

---

## 访问地址

- API 服务：`http://localhost:8080`
- Swagger 文档：`http://localhost:8080/boba/swagger/index.html`

## 关键配置文件

| 文件 | 说明 |
|------|------|
| `config/config.yml` | 开发环境配置（本地运行使用） |
| `config/config.production.yml` | 生产环境配置（Docker 使用） |
| `deployment/docker-compose.yml` | Docker Compose 编排文件 |
| `deployment/mysql/init/brewery.sql` | 数据库初始化 SQL |

---

## 停止服务

```bash
cd c2e-be-golang/deployment
docker-compose down
```
