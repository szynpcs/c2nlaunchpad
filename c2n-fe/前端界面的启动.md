# 前端界面的启动指南

## 环境要求

- Node.js
- Yarn
- MetaMask 浏览器插件
- 本地 Hardhat 节点（`c2n-contracts` 项目）

---

## 启动步骤

### 第一步：启动本地区块链节点

进入 `c2n-contracts` 目录，运行以下命令，**保持该终端窗口不要关闭**：

```bash
cd c2n-contracts
npx hardhat node
```

> 每次关闭节点重新启动后，链上所有合约状态会重置，需要重新执行第二步部署合约。

---

### 第二步：部署合约（首次启动或节点重启后必须执行）

新开一个终端，在 `c2n-contracts` 目录按顺序执行：

```bash
npx hardhat run scripts/deployment/deploy_c2n_token.js --network local
npx hardhat run scripts/deployment/deploy_airdrop_c2n.js --network local
npx hardhat run scripts/deployment/deploy_sales.js --network local
npx hardhat run scripts/deployment/deploy_farm.js --network local
```

> 部署完成后合约地址会自动写入 `deployments/contract-addresses.json`，前端会读取该文件。

---

### 第三步：启动前端

进入 `c2n-fe` 目录，运行：

```bash
cd c2n-fe
yarn dev
```

启动成功后访问：[http://localhost:3000](http://localhost:3000)

## 注意事项

### 关于 Claim C2N

- 每个地址只能 claim 一次，重复 claim 会报错 `Already claimed!`
- Airdrop 合约部署时会自动充值 10000 C2N，account[0] claim 后剩余 9900 C2N
- 必须使用 EOA 钱包（MetaMask），不支持合约钱包调用

### 关于支持的链

前端目前只支持以下两条链，连接其他链时合约功能不可用：

| 链 | Chain ID |
|---|---|
| 本地 Hardhat | 31337 |
| Sepolia 测试网 | 11155111 |

### 关于 Windows 启动

`package.json` 中的脚本已修改为使用 `cross-env` 处理环境变量，兼容 Windows，无需额外处理。

## 曾经修复过的问题

| 问题 | 文件 | 原因 |
|---|---|---|
| `yarn dev` 报 `export` 命令不存在 | `package.json` | Unix 语法不兼容 Windows，改用 `cross-env` |
| 页面加载崩溃 `invalid contract address` | `src/hooks/useWallet.ts` | 连接非配置链时地址为 `undefined`，加了 guard 防御 |
| 点击 Claim C2N 报错 | `src/containers/HomeBanner/HomeBanner.tsx` | 未连接钱包时缺少 `return`，导致继续执行合约调用 |
